# 夜之城信号 —— 部署指南

> 把整个夜之城搬上你的服务器，只需要一条命令。

本项目为 **全栈自包含** 设计：前端 + 后端 + MySQL 数据库全部打包在 Docker 中，
不依赖任何外部服务。支持任何安装了 Docker 的 Linux 服务器。

---

## 一、服务器准备（只需一次）

### 1. 安装 Docker

以 Ubuntu / Debian 为例：

```bash
curl -fsSL https://get.docker.com | bash
systemctl enable --now docker
```

其他发行版参考 Docker 官方文档。安装后执行 `docker compose version`
能输出版本号即表示就绪。

### 2. 上传项目代码

把整个项目文件夹上传到服务器，任选一种方式：

```bash
# 方式 A：本地打包上传（在项目目录的上一级执行）
tar -czf nightcity.tar.gz app/
scp nightcity.tar.gz root@你的服务器IP:/opt/

# 服务器上解压
ssh root@你的服务器IP
cd /opt && tar -xzf nightcity.tar.gz && cd app
```

```bash
# 方式 B：如果你会把代码推到 Git 仓库，直接 clone
git clone https://你的仓库地址.git app && cd app
```

## 二、修改管理员密码（重要）

默认管理员账号是 `admin / admin123`，**上线前必须修改**。

在服务器项目目录下创建 `.env` 文件（注意：这是给 docker compose 读取的，
与项目自带的本地开发 `.env` 不冲突，部署环境以此为准）：

```bash
cat > .env << 'EOF'
ADMIN_USERNAME=admin
ADMIN_PASSWORD=改成你的强密码
ADMIN_SECRET=改成一串足够长的随机字符
DB_ROOT_PASSWORD=改成数据库root密码
EOF
```

> ADMIN_SECRET 用于登录令牌签名，可用 `openssl rand -hex 32` 生成。

## 三、一键启动

```bash
docker compose up -d --build
```

首次启动会自动完成：构建镜像 → 启动 MySQL → 初始化数据表 → 写入示例文章 → 启动站点。

查看启动日志确认一切正常：

```bash
docker compose logs -f app
# 看到 "Server running on http://localhost:3000/" 即成功，Ctrl+C 退出日志
```

现在访问 `http://你的服务器IP:3000` 就能看到博客了。

> 如果服务器开了防火墙 / 云安全组，记得放行 **3000 端口**
> （配置好 Nginx 反代后则只需放行 80/443）。

## 四、绑定域名

1. 到你的域名服务商控制台，添加一条 **A 记录**：
   - 主机记录：`@`（或 `blog`、`www`）
   - 记录值：你的服务器公网 IP
2. 等 DNS 生效（通常几分钟）后，访问 `http://你的域名:3000`。

### （推荐）Nginx 反向代理 + 免费 HTTPS

让站点通过标准的 80/443 端口访问，并启用 HTTPS：

```bash
apt install -y nginx certbot python3-certbot-nginx
```

写入站点配置 `/etc/nginx/sites-available/nightcity`：

```nginx
server {
    listen 80;
    server_name 你的域名;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用并签发证书：

```bash
ln -sf /etc/nginx/sites-available/nightcity /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx
certbot --nginx -d 你的域名
```

完成后即可通过 `https://你的域名` 访问，证书会自动续期。

## 五、日常使用

| 操作 | 命令 / 入口 |
| --- | --- |
| 登录后台 | 访问 `/admin`，使用你设置的管理员账号 |
| 写文章 | 后台 → 新建文章（支持 Markdown，可预览） |
| 管理评论 | 后台 → 评论管理；或在文章页直接删除（登录状态下） |
| 查看日志 | `docker compose logs -f app` |
| 重启站点 | `docker compose restart app` |
| 停止站点 | `docker compose down` |
| 更新代码后 | `docker compose up -d --build` |

### 备份数据库（建议定期执行）

```bash
docker compose exec db mysqldump -uroot -p$DB_ROOT_PASSWORD nightcity > backup-$(date +%F).sql
```

数据本身保存在 Docker 卷 `db_data` 中，`docker compose down` 不会丢失；
但 `docker compose down -v` 会连卷一起删除，**慎用**。

## 六、常见问题

**Q: 打开后台提示 401 / 登录一直失败？**
检查 `.env` 中的 `ADMIN_USERNAME` / `ADMIN_PASSWORD` 是否与输入一致；
修改 `.env` 后需执行 `docker compose up -d` 重建容器生效。

**Q: 页面能打开但文章列表为空？**
首次启动时数据库还没就绪站点就已经开始初始化了，稍等几秒刷新即可。
如果持续为空，查看日志 `docker compose logs app` 确认初始化是否报错。

**Q: 想换掉示例文章？**
登录后台逐篇删除即可，种子文章只在「数据表为空」时写入一次。

**Q: 想改博客名字 / 页脚文案？**
修改 `src/components/Layout.tsx` 后重新 `docker compose up -d --build`。
